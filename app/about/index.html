About MiniWeb

“Sometimes, simple is best. MiniWeb is all about serving modern web content with classic C efficiency—no frills, no excess, just ultra-fast static sites for learning, hacking, or real-world deployment.”

Why MiniWeb?

  • Speed & Simplicity: No complex backend or frameworks—just serve your HTML, CSS, and static assets, instantly.
  • Tiny Footprint: Blazing fast and small, MiniWeb runs on anything from a Raspberry Pi to a big server.
  • Safe By Design: Serving static files means no headaches about state, injections, or authentication holes.
  • Learn & Hack: Dive into the C source to see how real sockets and protocols work, or use it as a base for your own server ideas!

How does it work?

MiniWeb listens for HTTP requests on your chosen port (default: 8080), then matches requested paths like /about.html to files in your chosen folder—delivering them with ultra-low overhead. Unknown routes get a friendly 404. It’s all clean C, cross-platform, and robust.

Get Started

  1. Clone the MiniWeb repository.
  2. Build it: make (requires a C compiler).
  3. Run it in your project folder: .bin/program
  4. Open http://localhost:8080/ in your browser.

Whether you’re learning, teaching, or deploying a tiny site, MiniWeb keeps things simple—and secure—so you can focus on building, not debugging.
